## 1. 브랜드 스토리

우리는 매일 수많은 정보 속에서 살아갑니다. 유튜브엔 재테크 영상이 넘치고, 주변엔 저마다 다른 말을 합니다. 하지만 정작 나에게 맞는 방법은 아무도 알려주지 않았습니다. 소득도 다르고, 나이도 다르고, 꿈도 다른데 왜 우리는 모두 같은 방법으로 돈을 모으려 할까요. 집 한 채, 노후 걱정 없는 삶, 가족과 여유로운 시간 — 이것은 특별한 사람만의 이야기가 아닙니다. 올바른 방향만 있다면 누구든 닿을 수 있는 곳입니다. 정답을 드리진 않습니다. 하지만 당신에게 맞는 길을 함께 찾겠습니다. 오늘 하루의 작은 습관이 10년 후의 당신을 만듭니다.

## 2. 기본 정보

- **앱 이름**: 메이트 (Mate)
- **슬로건(메인)**: 당신의 속도로, 당신의 미래로
- **슬로건(서브)**: 막막했던 돈 걱정, 이제 방향이 생깁니다
- **미션**: 모든 사람이 자신에게 맞는 방법으로 올바르게 소비·저축·투자하여 평생 안정감 있는 삶을 살 수 있도록 돕는다
- **비전**: 대한민국 모든 사람이 좋은 경제관념을 가지고 편안한 미래를 당연하게 그릴 수 있는 세상
- **핵심가치**: 나다움(Personalized) / 꾸준함(Sustainable) / 안심(Trustworthy)
- **콘텐츠 철학**: "이렇게 해라"가 아니라 "이해시킨다". 정답을 주는 앱이 아니라 스스로 정답을 찾을 수 있도록 돕는 앱.

## 3. 사업 목표 및 단계

```
목표: 토스 또는 네이버 같은 신뢰받는 플랫폼에 서비스로 입점/제안
1단계: Bubble.io로 웹 MVP 제작
2단계: 지인 대상 개념 검증 (필요성·타당성 확인) — 화면 목업으로 즉시 진행 가능
3단계: AI 생성 결과 다듬기 (1~2주)
4단계: SNS 콘텐츠 마케팅 (재테크 정보 콘텐츠로 신뢰 구축 → 앱 자연 노출)
5단계: 데이터 축적 (가입자 1,000명, MAU 40%+, 재방문 주2회+, NPS 40+)
6단계: Firebase + 전문 개발자로 정식 버전 전환
7단계: 토스/네이버 제안 (IR 자료 + 데이터)
```

## 4. 개발 도구 결정

- **현재 단계**: Bubble.io (노코드, AI 생성 기능 활용 중)
- **이유**: 코딩 경험 없음, 빠른 MVP 검증 필요, 예산 제한적
- **다음 단계**: Firebase + Claude Code (정식 버전 전환 시)
- **데이터 이전 방법**: Bubble CSV 내보내기 → Claude Code로 Firebase 형식 변환

## 5. 아이디어 보호 / 법적 사항

- 앱 이름 "메이트" 상표권 출원 필요 (kipris.or.kr, 약 5~10만원)
- 외부 개발자 협업 시 NDA 계약서 필수
- 투자 콘텐츠에 면책 고지 필수 ("교육 목적, 투자 책임은 본인")
- 은행 지출 자동 연동(오픈뱅킹)은 MVP 단계에서 제외, 사용자 직접 입력으로 시작

## 6. 데이터 수집 전략

- **정부지원사업**: 공공데이터포털(data.go.kr) API 연동 (MVP 단계는 직접 입력 데이터로 시작)
- **금융상품**: 금융위원회 오픈API (성장 단계에서 연동)
- **투자 정보**: 일부 API 가능, 상품 추천이 아닌 교육/정보 목적

---

## 7. 온보딩 설문 설계 (4단계)

1. **기본 정보**: 나이 / 직업 유형 / 거주 지역
2. **재무 현황**: 월 세후 수입 / 월 고정 지출 / 현재 저축액
3. **상황 파악**: 거주 형태 / 주택 보유 여부 / 결혼·자녀 여부
4. **목표 & 성향**: 재무 목표 / 저축 성향 / 투자 경험

→ 온보딩 완료 후 **점수(경제 건강점수) 표시하지 않음**. 대신 "OOO님이 지금 받을 수 있는 혜택" 카드 3장 즉시 제시.

## 8. 콘텐츠 전략 — 3대 핵심 섹션

### 8-1. 저축법
- 구조: 이해(왜 안 모이나) → 공감(나도 그랬어) → 원리(이래서 이렇게) → 사례
- 형식: 3초 스와이프 카드, 인스타 스토리 형식
- 사례는 "나와 비슷한 사람"의 감정 변화 중심 (숫자보다 공감)
- 숫자는 항상 "내 숫자"로 자동 변환해서 제시

### 8-2. 정부지원
- 구조: Before/After 카드 (현재 부담 → 지원 후 부담 → 차액 → 미래 누적 효과)
- 내 조건(나이/소득/거주지)으로 자동 필터링
- 어려운 용어를 쉬운 말로 번역 (예: 중위소득 250% → 4인가족 월소득 800만원)
- 신청 마감 D-day 표시
- 출처 명시 (예: 서민금융진흥원)로 신뢰도 확보

### 8-3. 투자
- 금융상품 추천 절대 금지 — 교육/이해 목적만
- 구조: 입문(투자가 뭔지 모르는 나) → 기초(투자 성향 테스트) → 실전준비(분산투자, 비상금 등)
- "오늘의 경제 한 줄": 사실 정보만 전달, "지금 사세요/투자하세요" 표현 금지
- 면책 고지 필수 배치

## 10. 경쟁 분석 핵심

- **토스**: UX 최강이지만 맞춤 코칭/교육 콘텐츠 부재 → 메이트는 이해 기반 경제관념 형성으로 차별화
- **뱅크샐러드**: 자산 분석 강하지만 정부지원 연동 없음 → 메이트는 3가지(저축법+정부지원+투자교육) 통합
- **카카오뱅크**: 간결한 UX의 정석 → 메이트도 "점수 없는 결과 화면"으로 부담 최소화
- **메이트 차별점**: 정보 통합 + 이해 우선 철학 + 점수 없는 결과 + 공감형 사례

## 14. 의사결정 원칙 (대표님 스타일)

- 정답을 주기보다 "이해시키는" 콘텐츠 철학 유지

- 일정/계획에 대해 막연히 동의하지 말고, 비판적·독립적으로 분석해서 맞다/아니다 명확히 답하기 (대표님이 직접 요청한 대화 방식)
