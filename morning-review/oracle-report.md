# Oracle Report — UX Drop-off 위험 지점 Top 3

분석 대상: `index.html`, `onboarding.html` (2026-06-16 기준)

## 1위 이탈 위험 — 온보딩 슬라이드 1: 스크롤 필요 인지 실패

**문제:** 슬라이드 1에 날짜 피커(184px) + 직업 선택(5개 라디오)이 함께 있어, 375px 화면에서 직업 선택 영역이 화면 밖으로 밀림. 사용자가 스크롤해야 한다는 걸 모르면 "다음" 버튼이 계속 비활성 상태인 이유를 알 수 없음.

**증거:** `next-btn:disabled`는 안내 메시지 없음. `checkSlide()`는 버튼을 끌 뿐, 왜 막혔는지 설명하지 않음.

**개선 방향:**
- 슬라이드 1을 두 개로 분리 (①생년월일 → ②직업)
- 또는 직업 미선택 시 버튼에 "직업을 선택해 주세요" 툴팁 표시

---

## 2위 이탈 위험 — 온보딩 슬라이드 2: 2단계 지역 선택 인지 실패

**문제:** 시/도 선택 후 시·군·구를 추가로 눌러야 "다음"이 활성화됨. 시/도만 누른 사용자는 "이미 선택했는데 왜 버튼이 안 눌리지?" 혼란.

**증거:** `REQ[2] = ['sido','sigungu']` — 둘 다 필요. 배지는 "시/도 선택됨 → 시·군·구 선택해주세요"로 바뀌지만, 이를 봤을 때는 이미 좌패널에서 손을 뗀 상태.

**개선 방향:**
- 시/도 선택 직후 우패널 상단에 강조 텍스트 "▶ 시·군·구를 선택해 주세요" 표시
- 또는 시/도만 선택해도 "다음" 일부 활성화 + 시·군·구는 선택적 처리

---

## 3위 이탈 위험 — index.html: 4카드 필수 시청 구조

**문제:** 첫 화면에서 온보딩으로 바로 가는 경로가 ①4번 카드까지 스와이프, ②우측 상단 "건너뛰기"뿐. 두 번째 경로는 12px 회색 텍스트라 모바일에서 잘 안 보임.

**증거:** "건너뛰기" `.skip-btn`은 `font-size: 14px; color: var(--color-gray1)` — 명도 낮고 탭 영역도 `padding: 8px`으로 좁음.

**개선 방향:**
- 건너뛰기 버튼 탭 영역을 최소 44×44px로 확대
- 또는 하단 "다음" 버튼 옆 보조 텍스트로 "건너뛰기"를 더 명확히 노출
