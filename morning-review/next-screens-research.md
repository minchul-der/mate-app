타깃: 22-38세 | 철학: 점수 없음, 길잡이 | 스택: HTML/CSS/JS

## 화면 3 — 맞춤 혜택 결과 (result.html)

### 레퍼런스 패턴 분석

**토스 (Toss)**
- 결과 화면에서 숫자(점수)보다 "지금 바로 신청 가능한 것" 리스트 우선 노출
- 카드 1장에 혜택명 + 한 줄 설명 + CTA 버튼 구조
- 조건 충족 여부를 초록/회색 뱃지로 간단 표시 (채점 느낌 없음)

**뱅크샐러드 (Banksalad)**
- "내 자산에 맞는 상품" 리스트를 Before/After 없이 카드 피드로 제공
- 각 카드에 예상 수익/혜택 금액을 크게 표시 → 즉각적 가치 체감
- 카드 상단 컬러 라인으로 카테고리 구분 (저축/정부지원/투자)

**카카오뱅크 (KakaoBank)**
- 사용자 이름 호명 ("홍길동님께 맞는 상품") → 개인화 느낌
- 가장 중요한 혜택 1개를 히어로 영역에 크게 강조
- 나머지는 슬림 카드 리스트로

### 메이트 최적 구조

```
[히어로] 사용자 상황 요약 태그 (3개) → "30대 초반 · 직장인 · 월세 거주자"
[카드 1] 가장 긴급한 혜택 (비상금→청년도약계좌, 내집마련→청약 등)
[카드 2] 중기 혜택 (소득·지역 기반 정부지원)
[카드 3] 장기 전략 (저축법 or 투자 공부 방향)
[CTA]   "자세히 알아보기" → 해당 콘텐츠 화면으로 이동
```

**핵심 원칙:**
- 점수/등급 표시 없음 — "당신의 상황"을 설명하는 말만
- 금액 수치로 가치 체감 ("월 최대 70만원 납입 가능")
- 각 카드에 "신청 조건 충족 여부"를 O/△ 로만 표시 (X 없음)

## 화면 4 — 저축법 콘텐츠 (saving.html)

### 레퍼런스 패턴 분석

**토스 피드**
- 짧은 카드 형식 아티클 (스와이프로 넘김)
- 제목 → 한 줄 요약 → 본문 2-3단락 → 실천 TIP 박스
- 길이: 1화면에 읽힐 수 있는 분량 (400자 이내)

**뱅크샐러드 레포트**
- Before/After 숫자 비교가 강력한 동기부여 도구
- 예: "월급 300만원, 이렇게 쓰면 vs 이렇게 바꾸면"
- 인터랙티브 슬라이더로 내 숫자를 직접 입력해보는 방식

**카카오 브런치 스타일**
- 시리즈 구조 (1편/2편/3편) → 완독률↑
- 각 편을 카드로 요약해서 "읽고 싶다"는 욕구 자극

### 메이트 최적 구조

```
[상단] 시리즈 제목 + 진행 도트 (1/4)
[본문 카드] 스와이프 카드 (좌우 넘기기)
  카드당 구조: 제목 → 핵심 포인트 1개 (크게) → 2-3줄 설명 → Before/After 수치
[하단] "다음 카드" + "전체 목차 보기"
```

**저축법 콘텐츠 4개 주제 (우선순위 순):**
1. 비상금 먼저 — "6개월치 생활비, 왜 필요한지"
2. 월급 쪼개기 — "통장 나누기의 실제 방법"
3. 자동이체 전략 — "의지 없이 모이는 구조 만들기"
4. 소비 패턴 파악 — "내가 어디에 새는지 알기"

## 화면 5 — 정부지원 상세 (benefit.html)

### 레퍼런스 패턴 분석

**토스 혜택 탭**
- 카테고리별 필터 (주거/청년/출산 등) → 탭 UI
- 각 항목: 혜택명 + 금액 + "내가 해당되는지" 체크 버튼
- 상세 페이지: 신청 방법을 단계별 스텝 (1→2→3)으로 표시

**정부24 / 복지로 (안 좋은 예)**
- 텍스트 나열, 조건 설명이 너무 복잡 → 이탈률 높음
- 메이트가 차별화할 지점: "내 조건에서 이 혜택이 맞는지"를 Yes/No로 미리 필터링

**뱅크샐러드 정부지원 화면**
- 사용자 프로필 기반으로 "신청 가능" / "조건 부족" 자동 분류
- Before/After: "지금 연 0원 → 이 혜택 받으면 연 840만원"

### 메이트 최적 구조

```
[상단] 내 조건 요약 칩 (나이·소득·지역·주거)
[필터] 전체 / 주거 / 소득 / 청년 / 출산·가족
[카드 리스트]
  각 카드: 혜택명 + 지원 금액 + 신청 가능 여부(✅/🔸) + "자세히" 버튼
[상세 모달]
  - 한 줄 요약
  - 신청 조건 (내 조건 대비 O/△ 표시)
  - 신청 방법 3단계
  - "신청하러 가기" (외부 링크)
```

## 화면 6 — 투자 공부 (learn.html)

### 레퍼런스 패턴 분석

**토스증권 주식 가이드**
- 개념 → 실제 예시 → 퀴즈 구조
- 짧은 단위 학습 (5분 완성)
- 진도 표시로 완료 욕구 자극

**카카오페이 증권 입문**
- 애니메이션 일러스트로 복잡한 개념을 시각화
- "예를 들어 1만원을 투자하면..."처럼 소액 기준으로 설명

### 메이트 최적 구조

```
[상단] 학습 단계 (입문 / 기초 / 심화)
[카드 스와이프] 각 개념 카드
  - 제목 + 이모지
  - 핵심 한 줄
  - 쉬운 비유 설명 (2-3줄)
  - "실제로는?" 예시
[퀴즈] OX 퀴즈로 이해 확인 (결과에 점수 없음 — "맞아요!" / "이렇게 생각하면 더 쉬워요!")
[진도] 학습한 카드 수 / 전체 수 (퍼센트 아님 — "3/8 완료")
```

**투자 공부 커리큘럼 (8개):**
1. 복리가 뭔지 — "72의 법칙"
2. ETF vs 개별주식 — "달걀과 바구니"
3. 분산투자 — "한 곳에 몰빵하지 않는 이유"
4. 시간분산 (적립식) — "타이밍보다 꾸준함"
5. 국내주식 세금 — "양도소득세 기초"
6. ISA계좌 — "절세의 기본"
7. 연금저축 — "노후 준비의 시작"
8. 리밸런싱 — "내 포트폴리오 관리법"

## 화면 간 흐름 (전체 앱 내비게이션)

```
index.html (인트로)
  ↓
onboarding.html (5단계 설문)
  ↓
result.html (맞춤 혜택 3장)
  ├── "저축법 배우기" → saving.html
  ├── "정부지원 상세보기" → benefit.html
  └── "투자 공부 시작" → learn.html
```

**공통 하단 탭 바 (화면 3부터 등장):**
홈(결과) / 저축법 / 정부지원 / 투자공부

---

## 개발 우선순위 추천

1. **result.html** — 온보딩과 연결되어 있어 MVP 검증에 필수. 내일 제작 가능.
2. **benefit.html** — 정부지원 조회가 메이트 핵심 가치. 정적 데이터 5개로 먼저 시작.
3. **saving.html** — 콘텐츠형이라 개발보다 글쓰기 비중이 높음.
4. **learn.html** — 마지막 단계. 퀴즈 로직이 추가로 필요해 개발 기간이 가장 긴.
